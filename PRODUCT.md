# Product

## Register

product

## Users

Audio developers, DSP engineers, plugin builders, and technically fluent musicians use this app on a local workstation while preparing paired dry/processed WAV files, training compact neural audio models, and validating exports for real-time RTNeural targets.

## Product Purpose

RTNeural Trainer turns paired audio captures into validated RTNeural JSON packages. Success means a user can import audio, prepare it, train a curated model, compare predictions, export the model, and trust the native validation and benchmark reports before moving into a plugin or embedded runtime.

## Brand Personality

Precise, calm, and workshop-like. The app should feel like a trustworthy bench tool for serious audio work: dense when needed, legible under repeated use, and confident without decorative noise.

## Anti-references

Avoid marketing-site composition, oversized hero sections, decorative gradients, soft SaaS dashboards, playful ML-toy styling, and generic card grids that hide operational detail. The interface should not look like a prompt demo, a cloud training portal, or a consumer audio preset browser.

## Design Principles

1. Put workflow state first: users should always know what is ready, blocked, running, or failed.
2. Expose proof, not promises: validation reports, benchmark results, package paths, and runtime details should be easy to inspect.
3. Prefer boring controls for serious work: standard inputs, buttons, tabs, tables, and inline panels beat novel affordances.
4. Keep the export target visible: RTNeural compatibility, backend choice, and device/runtime constraints should shape the UI.
5. Make recovery obvious: cancelled, interrupted, failed, and resumable states should be visible without digging through logs.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Preserve keyboard access for workflow tabs, forms, buttons, audio previews, and settings. Respect reduced-motion preferences. Do not rely on color alone for status; pair color with text labels and icons where possible.
